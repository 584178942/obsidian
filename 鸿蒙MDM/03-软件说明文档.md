## 一、项目概述

### 1.1 项目背景与目标

MDM 企业移动管理系统（鸿蒙 NEXT 6.0+）是由上海思御信息科技有限公司自主研发的企业级设备管控解决方案，适配华为鸿蒙 NEXT 6.0 及以上操作系统（API 14+）。系统通过 MQTT 协议建立设备与服务器之间的长连接通讯通道，实现企业对移动设备的远程集中管控，满足政企客户对设备安全合规管理的需求。

### 1.2 核心功能

- **设备远程控制**：锁定/解锁屏幕、恢复出厂设置、重启设备、重置密码
- **应用生命周期管理**：应用安装、卸载、获取已安装应用列表
- **网络管控**：WiFi 开关、移动数据开关、GPS 开关、蓝牙开关
- **外设限制**：相机、麦克风、USB 调试、截屏等禁用/启用
- **设备状态监控**：实时采集设备电量、充电状态、屏幕状态、网络类型
- **智能心跳**：根据电量/充电状态/屏幕状态动态调整心跳间隔
- **自动重连**：智能重连延迟策略，断线自动恢复

### 1.3 技术架构

| 层级 | 技术方案 |
|------|---------|
| 开发语言 | ArkTS（TypeScript 超集） |
| MQTT 客户端 | `@ohos/mqtt`（v5.0.0+） |
| MDM 能力 | `@kit.MDMKit`（Mock 当前模式） |
| 键值存储 | `@ohos.data.preferences` |
| 后台任务 | `@ohos.ability.backgroundTaskManager` |
| UI 框架 | ArkUI（声明式） |
| 日志 | `@kit.PerformanceAnalysisKit`（hilog） |

## 三、核心模块详细说明

### 3.1 MQTT 通信层（mqtt/）

**MqttClient.ets（约481行）**

MQTT客户端封装是整个系统的通信核心，负责与MQTT Broker建立并维持长连接。该模块实现了完整的MQTT客户端功能，包括连接管理、主题订阅、消息发布、连接状态监听、消息监听和智能重连。

模块定义了`MqttConnectionState`枚举，包含`DISCONNECTED`、`CONNECTING`、`CONNECTED`、`RECONNECTING`、`ERROR`五种连接状态，用于精确描述客户端当前状态。外部模块可通过`addConnectionListener`和`addMessageListener`注册监听器，在连接状态变化或消息到达时获得通知。

在连接管理方面，客户端在连接成功时自动订阅命令下发主题`mdm/{deviceId}`和配置更新主题`device/config/{deviceId}`，并设置遗嘱消息（Will Message）到`mdm/report`主题，当客户端异常断开时Broker自动发布离线通知。在发布功能上，模块封装了三个专用发布方法：`publishCommandResult`用于上报命令执行结果到`mdm/command/{deviceId}`主题，`publishHeartbeat`用于发送心跳到`mdm/{deviceId}/heartbeat`主题（QoS 0），`publishAlert`用于发送告警到`mdm/{deviceId}/alert`主题。

**MqttConfig.ets（约128行）**

配置参数管理模块，定义了所有MQTT通信相关的静态配置常量，包括服务器地址`tcp://192.168.1.199:1883`、客户端ID前缀`mdm_siyu_`、连接超时30秒、保活间隔60秒、QoS配置、遗嘱消息配置等。同时提供静态方法生成各类型主题（命令主题、配置主题、结果主题、心跳主题、告警主题）。

**ReconnectManager.ets（约97行）**

智能重连策略管理模块，实现了基于断开时长的渐进式重连延迟算法：断开5分钟内每5秒重试，5至30分钟每30秒重试，30至60分钟每60秒重试，1至6小时每3分钟重试，超过6小时每5分钟重试。通过记录断开时间戳动态计算下次重连延迟，在保障恢复速度的同时降低服务器连接压力。

### 3.2 命令处理系统（command/）

**CommandParser.ets（约110行）**

命令消息解析模块，`CommandMessage`类封装了命令消息的结构，包含`commandId`（命令唯一标识）、`action`（命令动作）、`params`（命令参数，可为null）和`timestamp`（时间戳）。`fromJson`静态方法将JSON字符串解析为CommandMessage对象，解析失败时返回null。提供`getString`、`getBoolean`、`getNumber`三个安全取值方法处理参数获取，避免空指针异常。

**CommandResult.ets（约102行）**

命令结果封装模块，`CommandResult`类封装了命令执行结果的结构，包含`commandId`、`action`、`success`（布尔值）、`message`（结果描述）、`data`（附加数据，可为null）和`timestamp`。提供`success`和`failure`两个静态工厂方法快速创建成功或失败结果，`toJson`方法将结果序列化为JSON字符串用于MQTT上报。

**CommandHandler.ets（约230行）**

命令处理分发核心模块，接收MQTT消息中的命令并根据`action`字段分发到对应的处理方法。所有处理方法均为`async`方法，支持异步操作。支持的命令涵盖设备控制（lock、unlock、wipe、reboot、reset_password）、应用管理（install_app、uninstall_app、get_app_list）、限制策略（disable_wifi、enable_wifi、disable_mobile_data、enable_mobile_data、set_gps、set_bluetooth、disable_camera、enable_camera、disable_microphone、enable_microphone、disable_usb_debug、enable_usb_debug、disable_screenshot、enable_screenshot）、信息查询（get_device_info）。未知命令返回包含`未知命令: {action}`的失败结果。

### 3.3 MDM 能力接口与工厂模式（mdm/）

**IDeviceController.ets（约175行）**

定义了四个MDM核心接口：`IDeviceController`（设备控制）、`IAppManager`（应用管理）、`IRestrictionManager`（限制策略）、`IInfoQuerier`（信息查询）。每个接口定义了对应的操作方法签名，返回`Promise<{success: boolean, message: string, data?: {...}}>`格式的结果对象，便于统一错误处理和日志记录。

**MdmApiFactory.ets（约113行）**

工厂类，是本系统架构设计的核心亮点。模块级单例模式，根据`USE_MOCK`标志自动选择Mock实现或真实MDM Kit实现。当前`USE_MOCK = true`（代码中TODO注释说明获取证书后改为false并取消注释真实实现）。这种设计使得开发阶段可以在没有企业MDM证书的情况下完成全部业务逻辑开发，商用时只需切换标志并补充真实实现即可。工厂类提供`getDeviceController()`、`getAppManager()`、`getRestrictionManager()`、`getInfoQuerier()`四个方法获取对应模块的单例实例，`reset()`方法用于测试或模式切换。

**Mock实现模块**

`DeviceControllerMock.ets`（约36行）：模拟设备锁定/解锁/擦除/重启/密码重置操作，返回固定成功响应和模拟延迟。

`AppManagerMock.ets`（约33行）：模拟应用安装/卸载/列表查询，返回模拟应用数据。

`RestrictionManagerMock.ets`（约51行）：模拟WiFi、移动数据、GPS、蓝牙、相机、麦克风、USB调试、截屏等限制操作的启用/禁用状态切换。

`InfoQuerierMock.ets`（约42行）：模拟设备信息查询，返回模拟的设备制造商、型号、操作系统版本等信息。

### 3.4 后台服务（service/）

**MqttBackgroundService.ets（约271行）**

后台长连接服务，是应用的核心后台组件，负责维护MQTT连接并协调各模块工作。

启动流程（`start`方法）：首先初始化`ConfigStore`配置存储，获取设备ID；创建`MqttClientManager`实例并初始化；注册连接状态监听器（`onConnectionStateChanged`）处理连接状态变化；注册消息监听器（`onMessageReceived`）处理收到的MQTT消息；申请后台运行权限（`requestBackgroundRunningprivilege`）；启动心跳定时器；连接MQTT服务器。

停止流程（`stop`方法）：停止心跳定时器、断开MQTT连接并释放资源、取消后台任务、设置运行标志为false。

消息处理（`onMessageReceived`方法）：解析JSON命令消息，调用`CommandHandler.handle`执行命令，通过`MqttClient.publishCommandResult`上报执行结果。

心跳机制（`startHeartbeat`方法）：使用`setTimeout`递归实现动态间隔心跳，每次发送前根据`HeartbeatAdjuster`重新计算下次间隔，首次立即发送，后续根据当前电量、充电状态、屏幕状态动态调整（1分钟/2分钟/5分钟/10分钟）。

### 3.5 心跳与电源管理（battery/）

**HeartbeatAdjuster.ets（约169行）**

动态心跳间隔调整模块，根据设备状态智能调整心跳上报频率。

定义了`HeartbeatMode`枚举：高性能模式（充电中且电量>50%，间隔60秒）、正常模式（默认，间隔120秒）、省电模式（电量<20%且未充电，间隔300秒）、极致省电模式（息屏且未充电，间隔600秒）。

`getMode()`方法综合电池电量、充电状态、屏幕状态判断当前模式，`buildHeartbeatMsg`方法构建包含设备ID、电量、充电状态、屏幕状态、网络类型、当前模式和 时间戳的心跳消息JSON。

**BatteryMonitor.ets（约132行）**

电池状态监控模块，封装`@ohos.battery` API，实时读取电池电量和充电状态。

### 3.6 网络监控（network/）

**NetworkMonitor.ets（约153行）**

网络状态监控模块，封装`@ohos.net.connection` API，实时获取网络连接状态和网络类型（WiFi/移动数据/无网络）。

### 3.7 配置存储（storage/）

**ConfigStore.ets（约283行）**

配置持久化管理模块，基于`@ohos.data.preferences`，存储和管理以下配置项：设备ID（自动生成并持久化）、MQTT服务器URI、MQTT用户名/密码、心跳间隔、最后心跳时间、管理员激活状态、Mock模式标志、最后同步时间。

`getDeviceId()`方法在首次调用时自动生成格式为`mdm_{时间戳36}_{随机字符}`的设备ID并持久化存储。

### 3.8 应用入口（entryability/）

**EntryAbility.ets（约72行）**

应用入口Ability，在`onCreate`中初始化`ConfigStore`；在`onWindowStageCreate`中加载主页`pages/Index`并调用`getMqttBackgroundService().start()`启动后台MQTT长连接服务；在`onDestroy`中调用`service.stop()`停止后台服务。

### 3.9 UI 页面（pages/）

**Index.ets（约399行）**

应用主页，展示设备实时状态并提供命令测试功能。页面顶部为设备状态卡片，显示设备ID、MQTT连接状态、电池电量（含进度条和颜色指示）、网络类型、心跳模式。中间为MQTT命令测试区域，以网格按钮形式展示可测试的命令（lock、unlock、reboot、get_device_info、disable_wifi、enable_wifi、disable_camera、enable_camera），下方为命令执行日志显示区域。底部为MDM能力说明，提示当前运行在Mock模式及切换到真实MDM能力的方法。页面通过`setInterval`每2秒刷新一次设备状态。

**Settings.ets（约313行）**

设置页面，提供本地配置管理功能。

## 四、MQTT 消息协议

### 4.1 主题定义

| 主题 | 通信方向 | 用途 |
|------|---------|------|
| `mdm/{deviceId}` | 服务器 → 设备 | 下发管控指令 |
| `device/config/{deviceId}` | 服务器 → 设备 | 下发配置更新 |
| `mdm/report` | 设备 → 服务器 | 遗嘱（离线通知） |
| `mdm/command/{deviceId}` | 设备 → 服务器 | 上报命令执行结果 |
| `mdm/{deviceId}/heartbeat` | 设备 → 服务器 | 设备心跳数据（QoS 0） |
| `mdm/{deviceId}/alert` | 设备 → 服务器 | 设备告警信息 |

### 4.2 命令消息格式

```json
{
  "commandId": "uuid-123456",
  "action": "lock",
  "params": {
    "mainText": "设备已锁定",
    "subText": "请联系管理员"
  },
  "timestamp": 1704067200000
}
```

### 4.3 支持的命令

| action | 说明 | 必要参数 |
|--------|------|---------|
| lock | 锁定设备屏幕 | mainText, subText |
| unlock | 解锁屏幕 | - |
| wipe | 恢复出厂设置 | - |
| reboot | 重启设备 | - |
| reset_password | 重置密码 | password |
| install_app | 安装应用 | apkPath |
| uninstall_app | 卸载应用 | bundleName |
| get_app_list | 获取应用列表 | - |
| disable_wifi / enable_wifi | WiFi禁用/启用 | - |
| disable_mobile_data / enable_mobile_data | 移动数据禁用/启用 | - |
| set_gps | GPS开关 | enabled |
| set_bluetooth | 蓝牙开关 | enabled |
| disable_camera / enable_camera | 相机禁用/启用 | - |
| disable_microphone / enable_microphone | 麦克风禁用/启用 | - |
| disable_usb_debug / enable_usb_debug | USB调试禁用/启用 | - |
| disable_screenshot / enable_screenshot | 截屏禁用/启用 | - |
| get_device_info | 查询设备信息 | - |

## 五、错误处理

### 5.1 错误码定义

| 错误码 | 说明 |
|--------|------|
| 201 | 权限验证失败 |
| 401 | 参数错误 |
| 9200001 | 应用未激活设备管理员 |
| 9200002 | 管理员无权限执行该操作 |
| 1001 | MQTT连接失败 |
| 1002 | MQTT发布失败 |
| 1003 | MQTT订阅失败 |
| 2001 | 命令解析错误 |
| 2002 | 命令执行错误 |
| 3001 | 配置存储初始化失败 |

### 5.2 错误处理策略

MQTT连接失败时触发智能重连；命令解析失败时记录错误日志并返回失败结果；MDM接口调用失败时返回详细错误信息；存储操作失败时使用内存缓存降级处理。
