## 二、系统总体架构

### 2.1 架构设计理念

本系统采用**厂商抽象层（Vendor Abstraction）** 架构设计，核心思想是将各Android设备厂商的MDM实现细节封装在统一接口背后。上层业务逻辑不直接依赖具体厂商实现，而是通过工厂模式自动检测当前设备厂商并注入对应的MDM管理器。

这种设计实现了：
- **厂商无关**：业务逻辑与具体厂商解耦，新增厂商只需实现接口
- **可测试性**：可通过Mock厂商实现进行单元测试
- **可扩展性**：新增厂商只需添加新的实现类，无需修改上层代码
- **代码复用**：通用逻辑在上层抽象接口中定义，避免重复实现

## 三、核心模块详解

### 3.1 MQTT通信模块

**文件包**：`com.siyu.mdm.enterprise.util.mqtt`

#### MqttManager（MQTT客户端封装）

MqttManager是MQTT通信的核心封装类，负责与MQTT Broker建立并维护长连接。

**核心功能**：
- MQTT客户端初始化与连接管理
- 自动重连机制（指数退避策略，首次10秒，最大5分钟）
- 主题订阅与取消订阅
- 消息发布（QoS 1保证）
- 遗嘱消息（Will）配置，设备异常离线时自动发布
- 连接状态回调监听

**连接参数**：
| 参数 | 值 | 说明 |
|------|-----|------|
| 服务器地址 | `tcp://192.168.1.199:1883` | MQTT Broker地址 |
| 连接超时 | 30秒 | TCP连接建立超时 |
| 心跳间隔 | 60秒 | 保活探测间隔 |
| QoS | 1 | At Least Once |
| 遗嘱主题 | `mdm/report` | 设备离线状态发布 |

#### MqttTopicManager（主题管理）

统一管理MQTT主题命名规范：

| 主题 | 方向 | 用途 |
|------|------|------|
| `mdm/{deviceId}` | 服务器→设备 | 设备私有命令下发 |
| `mdm/report` | 设备→服务器 | 设备状态/遗嘱消息 |
| `mdm/command/{deviceId}` | 设备→服务器 | 命令执行结果上报 |
| `mdm/{deviceId}/heartbeat` | 设备→服务器 | 设备心跳上报 |
| `mdm/{deviceId}/alert` | 设备→服务器 | 告警信息上报 |

#### MqttMessageBuilder（消息构建）

提供各类MQTT消息的JSON构造能力，包括心跳消息、命令结果消息、告警消息的格式化封装。

#### MqttPowerManager（电源管理）

与系统电源管理交互，负责在低电量/熄屏等场景下协调MQTT连接策略。

#### MqttCommandHandler（命令处理器）

消息到达后的第一层处理，负责JSON解析、消息类型判断，然后分发给MdmCommandManager执行。

### 3.3 厂商抽象层

#### MDMManagerFactory（工厂类）

自动检测设备厂商并提供对应的MDM管理器实例，是整个抽象层的核心入口。

**厂商检测逻辑**：

```kotlin
when (Build.MANUFACTURER.lowercase()) {
    contains("huawei") || contains("honor") -> Vendor.HUAWEI
    contains("xiaomi") || contains("redmi") -> Vendor.XIAOMI
    contains("vivo") -> Vendor.VIVO
    else -> Vendor.UNKNOWN
}
```

**工厂方法**：

```kotlin
MDMManagerFactory.getSecurityManager()   // 获取安全管理器
MDMManagerFactory.getAppManager()        // 获取应用管理器
MDMManagerFactory.getNetworkManager()    // 获取网络管理器
MDMManagerFactory.getHardwareManager()   // 获取硬件管理器
MDMManagerFactory.isSupported()          // 检查是否支持MDM
```

#### Vendor枚举

定义支持的设备厂商：

| 枚举值 | 厂商 | 说明 |
|--------|------|------|
| `HUAWEI` | 华为/荣耀 | 完整实现 |
| `XIAOMI` | 小米/红米 | 完整实现 |
| `VIVO` | vivo | 完整实现 |
| `UNKNOWN` | 未知 | 不支持，抛出异常 |

#### 接口定义（AppManager / HardwareManager / NetworkManager / SecurityManager）

四大Manager接口定义了设备管理能力的抽象规范：

**AppManager（应用管理接口）**：
- `install(apkPath: String)` — 安装应用
- `uninstall(packageName: String)` — 卸载应用
- `setInstallBlacklist(pkgs: Set<String>)` — 设置安装黑名单
- `setUninstallBlacklist(pkgs: Set<String>)` — 设置卸载黑名单
- `getInstalledApps()` — 获取已安装应用列表

**HardwareManager（硬件控制接口）**：
- `setCameraDisabled(disabled: Boolean)` — 摄像头控制
- `setMicrophoneDisabled(disabled: Boolean)` — 麦克风控制
- `setScreenCaptureDisabled(disabled: Boolean)` — 截屏控制
- `setUSBDataDisabled(disabled: Boolean)` — USB数据控制
- `setBluetoothDisabled(disabled: Boolean)` — 蓝牙控制

**NetworkManager（网络管理接口）**：
- `setWifiDisabled(disabled: Boolean)` — WiFi控制
- `setMobileDataDisabled(disabled: Boolean)` — 移动数据控制
- `setNFCEnabled(enabled: Boolean)` — NFC控制
- `setGPSEnabled(enabled: Boolean)` — GPS控制

**SecurityManager（安全管理接口）**：
- `lock(mainText, subText)` — 锁定设备
- `unlock()` — 解锁设备
- `wipe()` — 恢复出厂设置
- `reboot()` — 重启设备
- `resetPassword(password)` — 重置密码

### 3.4 华为MDM实现层

**文件包**：`com.siyu.mdm.enterprise.util.mdm.huawei`

华为实现是当前系统中功能最完整的厂商适配，充分利用华为MDM Kit企业接口。

#### HuaweiAdminManager（管理员激活）

- `activateAdmin(reason: String)` — 请求激活设备管理员
- `isDeviceAdmin(): Boolean` — 检查是否为管理员
- `getDiagnosticReport(): String` — 获取诊断报告

#### HuaweiSecurityManager（华为安全实现）

继承SecurityManager接口，封装华为DevicePolicyManager能力：
- 设备锁定/解锁：通过`lock()`调用全屏锁屏Activity
- 恢复出厂：`wipeData(0)`
- 密码重置：`resetPassword()`

#### HuaweiAppManager（华为应用实现）

封装华为`DeviceApplicationManager`和`DevicePackageManager`：
- 安装：`installPackage()`
- 卸载：`uninstallPackage()`
- 安装黑名单：`addInstallPackageBlockList()`
- 运行管控：`setTaskLockAppList()` / `addDisallowedRunningApp()`

#### HuaweiNetworkManager（华为网络实现）

封装华为`DeviceNetworkManager`：
- WiFi控制：`setWifiDisabled()`
- 移动数据：`DeviceControlManager.turnOnMobiledata()`
- APN管理：`addApn()` / `deleteApn()`

#### HuaweiHardwareManager（华为硬件实现）

封装华为`DeviceCameraManager`和反射调用`DeviceRestrictionManager`：
- 摄像头：`setVideoDisabled()`
- 截屏：`setScreenCaptureDisabled()`
- 麦克风：`setMicrophoneDisabled()`
- USB：`setUSBDataDisabled()` / `setUSBOtgDisabled()`
- 蓝牙：通过反射调用`setBluetoothDisabled()`
- NFC：通过反射调用`setNFCDisabled()`
- 多窗口、安全模式、文件分享等：通过DeviceRestrictionManager反射调用

## 五、前台服务（MqttService）

**文件**：`com.siyu.mdm.enterprise.service.MqttService`

作为Android前台Service运行，是设备与服务器保持长连接的核心服务。

**服务职责**：
- 初始化MQTT客户端并建立连接
- 订阅设备私有命令主题
- 接收并解析MQTT消息，转发给MdmCommandManager
- 将命令执行结果通过MQTT回传
- 维护设备在线状态

**前台通知**：
- 创建通知渠道，固定显示"MQTT设备服务运行中"
- 服务异常断开时自动重连
- Android 13+适配通知权限，Android 14+适配前台服务类型

## 十、技术特点总结

1. **多厂商抽象架构**：通过工厂模式和接口抽象，优雅支持华为/小米/vivo三大主流Android设备厂商，新增厂商成本极低
2. **MQTT轻量长连接**：基于Eclipse Paho实现稳定MQTT通信，支持遗嘱消息、自动重连（指数退避）
3. **命令可追溯**：每条命令携带唯一commandId，支持批量命令并行执行与结果汇总上报
4. **厂商功能完整性**：华为实现覆盖30余项设备控制能力，充分利用华为MDM Kit能力
5. **智能心跳策略**：根据设备状态（亮屏/熄屏/充电/低电量）动态调整心跳间隔，兼顾保活与省电
6. **前台服务保活**：MqttService以foregroundService运行，防止被系统Kill导致断连
7. **反射安全调用**：对部分设备特定API使用反射调用，避免因API不存在导致崩溃
8. **诊断工具完善**：提供各管理器的诊断报告方法，便于排查设备兼容性问题
