**版本**：V2.1

**适用平台**：HarmonyOS Next 6.0+  
**应用包名**：`com.siyu.mdm`

## 一、产品概述

MDM 企业移动管理系统（鸿蒙6.0版）是上海思御信息科技有限公司自主研发的企业级移动设备管控应用，运行于华为鸿蒙 NEXT 6.0 及以上操作系统设备上，通过 MQTT 协议与服务器建立长连接，实现企业移动设备的远程集中管控。

系统提供设备远程锁定、应用管理、网络与外设管控、设备信息查询等功能，适用于政企单位对移动设备的合规安全管理需求。

## 二、安装与激活

### 2.1 安装应用

1. 打开设备自带的华为应用市场（AppGallery），搜索"SiYuMDM"或"MDM企业管理系统"；
2. 点击安装，等待应用下载并自动安装完成；
3. 安装完成后，在应用列表中找到并打开应用。

### 2.2 激活设备管理员

首次打开应用时，系统会引导用户激活设备管理员权限，这是 MDM 应用正常工作所必须的安全权限。

**激活步骤：**

1. 打开应用后，界面显示"设备管理员激活请求"对话框；
2. 用户阅读并确认管理员权限说明，确认后点击"激活"按钮；
3. 系统调用 `enterpriseAdminManager.activateAdmin` 接口激活 EnterpriseAdminAbility；
4. 激活成功后，界面显示"激活成功"提示，应用进入主界面；
5. 若激活失败（报错9200001），请检查设备安全设置是否允许安装设备管理员应用。

**取消激活（禁用管理员）：**

1. 进入应用设置页面，找到"取消激活设备管理员"选项；
2. 点击确认后，系统禁用 EnterpriseAdminAbility，设备恢复普通设备状态；
3. 取消激活后，远程管控指令将无法执行。

## 三、界面与功能

### 3.1 主界面（Index.ets）

主界面展示当前设备的实时状态信息，包括：

- **设备信息卡片**：显示设备型号、制造商、操作系统版本、SDK版本
- **连接状态指示**：显示 MQTT 服务连接状态（在线/离线），在线时显示绿色标识
- **电池状态**：显示当前电量、是否充电、屏幕开关状态
- **网络类型**：显示当前网络连接方式（WiFi/移动数据/无网络）
- **最近命令记录**：显示最近一次执行的管控命令及执行结果

### 3.2 设置页面（Settings.ets）

设置页面提供本地配置管理功能：

- **服务器地址配置**：查看 MQTT Broker 地址（默认：tcp://192.168.1.199:1883）
- **心跳间隔**：查看当前心跳策略模式（高性能/普通/省电/超省电）
- **设备ID查看**：查看本设备的唯一标识符
- **日志开关**：开启或关闭本地日志记录
- **关于**：查看应用版本号、开发者信息、用户协议

## 四、远程管控操作指南

以下操作由服务器管理员通过 MDM 管理平台（网页或服务端接口）下发指令执行，设备端自动接收并执行。

### 4.1 设备远程锁定

**操作说明**：远程锁定设备屏幕，显示管理员指定的提示信息，防止未授权人员使用设备。

**服务器下发命令示例**：
```json
{
  "commandId": "cmd-lock-001",
  "action": "lock",
  "params": {
    "mainText": "设备已锁定",
    "subText": "请联系管理员：400-xxx-xxxx"
  },
  "timestamp": 1704067200000
}
```

**执行效果**：
- 设备屏幕立即锁定，显示"设备已锁定"主提示和联系管理员的副提示；
- 设备所有触摸、按键操作被拦截，无法进入系统；
- 解锁需联系管理员发送 unlock 指令。

**异常处理**：若设备未激活管理员权限，此命令执行失败并返回错误码9200001。

### 4.2 设备远程解锁

**服务器下发命令示例**：
```json
{
  "commandId": "cmd-unlock-001",
  "action": "unlock",
  "timestamp": 1704067200000
}
```

**执行效果**：设备屏幕立即解锁，恢复正常操作状态。

### 4.3 恢复出厂设置（wipe）

**操作说明**：清除设备上所有用户数据和应用，恢复到出厂状态。**此操作不可逆，请谨慎使用。**

**服务器下发命令示例**：
```json
{
  "commandId": "cmd-wipe-001",
  "action": "wipe",
  "timestamp": 1704067200000
}
```

**执行效果**：
- 设备立即停止所有应用和服务；
- 清除所有用户数据、账户、设置和应用；
- 设备恢复到出厂时的初始状态。

### 4.4 设备远程重启

**服务器下发命令示例**：
```json
{
  "commandId": "cmd-reboot-001",
  "action": "reboot",
  "timestamp": 1704067200000
}
```

**执行效果**：设备在收到命令后立即执行重启，适用于固件更新或系统配置刷新后需要重启生效的场景。

### 4.5 远程重置密码

**服务器下发命令示例**：
```json
{
  "commandId": "cmd-resetpwd-001",
  "action": "reset_password",
  "params": {
    "password": "Admin@1234"
  },
  "timestamp": 1704067200000
}
```

**执行效果**：设备解锁密码被修改为指定新密码，下次解锁时需使用新密码。

### 4.6 应用安装

**操作说明**：远程安装企业内测应用，支持指定 APK 路径进行静默安装（无需用户交互）。

**服务器下发命令示例**：
```json
{
  "commandId": "cmd-install-001",
  "action": "install_app",
  "params": {
    "apkPath": "/data/local/tmp/enterprise_app.apk"
  },
  "timestamp": 1704067200000
}
```

**执行效果**：
- 系统在后台下载并安装指定路径的 APK 文件；
- 安装过程无需用户操作；
- 安装完成后自动将应用添加到设备应用列表。

**前置条件**：设备必须已激活管理员权限，否则安装失败。

### 4.7 应用卸载

**服务器下发命令示例**：
```json
{
  "commandId": "cmd-uninstall-001",
  "action": "uninstall_app",
  "params": {
    "bundleName": "com.example.unwanted_app"
  },
  "timestamp": 1704067200000
}
```

**执行效果**：指定包名的应用被强制卸载，应用数据和缓存一并清除。

## 五、在线状态与心跳机制

### 5.1 在线状态

设备通过 MQTT 长连接与服务器保持通信。连接状态由以下因素决定：

- MQTT 客户端与 Broker 之间 TCP 连接是否正常
- 保活心跳（Keep-Alive）是否按时发送
- 设备网络是否可用（WiFi/移动数据是否开启）

### 5.2 心跳上报机制

设备按智能调整后的间隔向服务器上报心跳消息，包含设备当前状态：

| 字段 | 说明 |
|------|------|
| deviceId | 设备唯一标识 |
| batteryLevel | 电量百分比（0-100） |
| isCharging | 是否正在充电 |
| isScreenOn | 屏幕是否亮起 |
| networkType | 当前网络类型（WiFi/Mobile/None） |
| timestamp | 消息时间戳 |

**心跳间隔智能调整策略**：

| 条件 | 心跳间隔 |
|------|---------|
| 充电中且电量>50% | 1分钟 |
| 正常使用（屏幕亮） | 2分钟 |
| 电量<20%且未充电 | 5分钟 |
| 息屏且未充电 | 10分钟 |

### 5.3 离线通知

当设备与 MQTT Broker 的连接断开时（无论主动断开还是异常断开），Broker 自动向 `mdm/report` 主题发布设备的离线通知（Will Message），服务器据此将设备状态标记为离线。

## 六、异常处理

### 6.1 常见错误码

| 错误码 | 含义 | 处理建议 |
|--------|------|---------|
| 9200001 | 应用未激活设备管理员 | 在应用内重新激活管理员权限 |
| 9200002 | 管理员无权限执行该操作 | 检查 MDM 资质是否有效 |
| 201 | 权限验证失败 | 重新激活管理员权限 |
| 401 | 命令参数错误 | 检查下发命令的参数格式是否正确 |

### 6.2 网络异常处理

- **网络断开**：设备自动进入离线状态，心跳停止发送；网络恢复后，MQTT 客户端自动重连，恢复在线状态。
- **MQTT 连接失败**：按照智能重连策略（5秒→30秒→60秒→180秒→300秒）逐步延长重试间隔，降低服务器压力。
- **消息发送失败**：MQTT 发布操作在未连接状态下返回 false，消息不会丢失（QoS 1保障），网络恢复后可继续发送。

### 6.3 电池电量耗尽

当设备电量极低（<5%）即将关机时，系统可能强制终止后台服务，导致 MQTT 连接断开。设备关机重启后，应用自动重新连接 MQTT 服务器，无需人工干预。
