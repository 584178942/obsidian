## 1. 项目概述

### 1.1 项目背景

为企业提供基于鸿蒙系统的设备远程管控解决方案，通过 MQTT 长连接实现云端指令下发与状态上报。

### 1.2 目标平台

| 项目 | 版本 |
|------|------|
| HarmonyOS | Next 6.0+ |
| API Level | API 14+ |
| 语言 | ArkTS (TypeScript 超集) |

### 1.3 核心功能

| 功能类别 | 具体功能 |
|----------|----------|
| **设备控制** | 远程锁定/解锁、恢复出厂设置、重启、重置密码 |
| **应用管理** | 应用安装/卸载、黑名单管理 |
| **网络管控** | WiFi、移动数据、蓝牙、GPS 开关控制 |
| **外设限制** | 相机、麦克风、USB、截图等禁用 |
| **状态监控** | 电量、网络、在线状态实时上报 |
| **通信机制** | MQTT 长连接 + 智能心跳 + 自动重连 |

### 5.2 MqttClientManager 客户端接口

```typescript
// 文件: mqtt/MqttClient.ets

import mqtt from '@ohos/mqtt'

export class MqttClientManager {
  // 设备ID
  deviceId: string = ''

  // 连接状态
  isConnected: boolean = false

  // 连接状态监听器
  connectionListeners: Array<(connected: boolean) => void> = []

  // 消息监听器
  messageListeners: Array<(topic: string, payload: string) => void> = []

  /**
   * 初始化MQTT客户端
   * @param deviceId 设备唯一标识
   */
  initialize(deviceId: string): void

  /**
   * 连接MQTT服务器
   */
  connect(): void

  /**
   * 断开连接
   */
  disconnect(): void

  /**
   * 订阅主题
   * @param topic 主题名
   * @param qos 服务质量等级
   */
  subscribe(topic: string, qos?: number): void

  /**
   * 发布消息
   * @param topic 主题名
   * @param message 消息内容
   * @param qos 服务质量等级
   * @returns 是否发送成功
   */
  publish(topic: string, message: string, qos?: number): boolean

  /**
   * 添加连接状态监听器
   * @param listener 监听回调
   */
  addConnectionListener(listener: (connected: boolean) => void): void

  /**
   * 添加消息监听器
   * @param listener 监听回调
   */
  addMessageListener(listener: (topic: string, payload: string) => void): void

  /**
   * 释放资源
   */
  release(): void
}
```

## 6. 数据库设计

### 6.1 命令历史表 (SQLite)

**表名**: `command_history`

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 自增ID |
| `commandId` | TEXT | UNIQUE NOT NULL | 命令唯一标识 |
| `action` | TEXT | NOT NULL | 命令动作 |
| `params` | TEXT | | 命令参数(JSON) |
| `status` | TEXT | NOT NULL | 状态(running/success/failure) |
| `result` | TEXT | | 执行结果 |
| `timestamp` | INTEGER | NOT NULL | 时间戳 |
| `deviceId` | TEXT | NOT NULL | 设备ID |

**建表语句**:
```sql
CREATE TABLE IF NOT EXISTS command_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  commandId TEXT UNIQUE NOT NULL,
  action TEXT NOT NULL,
  params TEXT,
  status TEXT NOT NULL DEFAULT 'running',
  result TEXT,
  timestamp INTEGER NOT NULL,
  deviceId TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_command_timestamp ON command_history(timestamp DESC);
CREATE INDEX IF NOT EXISTS idx_command_deviceId ON command_history(deviceId);
```

## 7. 消息协议

### 7.1 MQTT 主题规划

| 主题 | 方向 | QoS | 说明 |
|------|------|-----|------|
| `mdm/{deviceId}` | 服务端→设备 | 1 | 接收指令 |
| `device/config/{deviceId}` | 服务端→设备 | 1 | 接收配置更新 |
| `mdm/report` | 设备→服务端 | 1 | 遗嘱消息(离线通知) |
| `mdm/command/{deviceId}` | 设备→服务端 | 1 | 命令执行结果 |
| `mdm/{deviceId}/heartbeat` | 设备→服务端 | 0 | 心跳数据 |
| `mdm/{deviceId}/alert` | 设备→服务端 | 1 | 告警信息 |

### 7.2 命令消息格式 (服务端→设备)

```json
{
  "commandId": "uuid-123456",
  "action": "lock",
  "params": {
    "mainText": "设备已锁定",
    "subText": "请联系管理员"
  },
  "timestamp": 1704067200000
}
```

### 7.3 命令结果消息格式 (设备→服务端)

```json
{
  "commandId": "uuid-123456",
  "action": "lock",
  "success": true,
  "message": "设备已锁定",
  "timestamp": 1704067201000,
  "data": {}
}
```

### 7.4 心跳消息格式

```json
{
  "deviceId": "device_xxx",
  "batteryLevel": 85,
```

## 9. 错误处理

### 9.1 错误码定义

| 错误码 | 常量名 | 说明 |
|--------|--------|------|
| 201 | `PERMISSION_DENIED` | 权限验证失败 |
| 401 | `INVALID_PARAMS` | 参数错误 |
| 9200001 | `NOT_DEVICE_ADMIN` | 应用不是设备管理员 |
| 9200002 | `ADMIN_NO_PERMISSION` | 管理员无权限执行该操作 |
| 1001 | `MQTT_CONNECTION_FAILED` | MQTT连接失败 |
| 1002 | `MQTT_PUBLISH_FAILED` | MQTT发布失败 |
| 1003 | `MQTT_SUBSCRIBE_FAILED` | MQTT订阅失败 |
| 2001 | `COMMAND_PARSE_ERROR` | 命令解析错误 |
| 2002 | `COMMAND_EXECUTE_ERROR` | 命令执行错误 |
| 3001 | `STORAGE_INIT_ERROR` | 存储初始化失败 |

### 9.2 错误处理策略

| 场景 | 处理策略 |
|------|----------|
| MQTT连接失败 | 触发重连,延迟指数增长 |
| 命令解析失败 | 记录错误日志,返回失败结果 |
| MDM接口调用失败 | 返回详细错误信息 |
| 存储操作失败 | 降级处理,使用内存缓存 |
| 网络不可用 | 等待网络恢复后重连 |
