## 1. 软件概述

MDM设备端应用是部署在Android设备上的客户端程序，配合MDM管理平台实现对企业移动设备的远程管控。

**主要功能：**

- 接收并执行管理平台下发的控制指令
- 设备状态监控与心跳上报
- 应用安装、卸载管控
- 硬件功能（WiFi/蓝牙/USB/摄像头等）远程控制
- 设备锁屏、解锁、恢复出厂设置
- 机卡绑定安全校验
- 锁屏界面自定义

## 2. 安装与激活

### 2.1 安装前准备

- 设备需为Android 7.0及以上版本
- 设备需支持华为MDM接口（建议使用华为设备）
- 获得管理平台分配的设备注册码

### 2.2 安装步骤

**Step 1**：获取安装包

从管理员处获取APK安装包文件，拷贝至设备存储。

**Step 2**：允许安装未知来源应用

> 设置 → 安全与隐私 → 更多设置 → 外部来源应用下载 → 允许（若设备已受MDM管控，此步骤可能由管理员直接配置）

**Step 3**：安装APK

找到APK文件，点击安装，按提示完成安装。

**Step 4**：激活设备管理员

首次打开应用时，系统会提示"激活设备管理员"，选择**允许**。

> ⚠️ 若不激活设备管理员，MDM核心功能将无法使用。

**Step 5**：等待注册成功

应用激活后会自动连接管理平台，完成设备注册。注册成功后状态栏显示通知"设备服务运行中"。

### 2.3 激活失败处理

| 现象 | 原因 | 处理方法 |
|------|------|---------|
| 无法激活设备管理员 | 设备已有其他管理员 | 移除旧的管理应用或恢复出厂设置 |
| 注册连接失败 | 网络不通 | 检查WiFi/移动数据是否正常 |
| 显示"连接超时" | MQTT服务器地址错误 | 联系管理员确认服务器配置 |
| 应用频繁退出 | 权限不足 | 检查通知权限、前台服务权限是否授予 |

## 3. 主界面介绍

应用主界面（MainActivity）用于展示当前设备基本信息与连接状态。

### 3.1 状态指示

| 区域 | 显示内容 |
|------|---------|
| 顶部状态栏 | 设备服务运行通知（常驻前台服务） |
| 连接状态 | MQTT连接状态（已连接/未连接/重连中） |
| 设备信息 | 设备型号、IMEI、当前SIM卡ICCID |
| 版本信息 | 当前应用版本号 |

### 3.2 通知栏标识

应用作为前台服务运行，在通知栏显示"MQTT设备服务运行中"，表示服务正常运行。

## 4. 设备控制操作

以下操作通过管理平台远程下发指令执行，设备端无需人工干预。

### 4.1 设备锁定（lock）

**指令效果：**
- 设备立即进入全屏锁屏状态
- 显示管理员自定义文字（默认："终端强制锁定，如需解锁请与管理员联系"）
- 设备无法正常使用，需管理员下发解锁指令

**触发方式：** 管理平台下发 `lock` 指令

### 4.2 设备解锁（unlock）

**指令效果：**
- 退出单应用锁屏模式
- 恢复正常设备使用

**触发方式：** 管理平台下发 `unlock` 指令

### 4.3 应用安装（installUpdateApp）

**指令效果：**
- 设备自动下载指定APK文件
- 自动安装或更新应用
- 支持企业微信等办公应用静默安装

**触发方式：** 管理平台下发 `installUpdateApp` 指令，携带APK下载地址

### 4.4 应用卸载（uninstallApp）

**指令效果：**
- 远程卸载指定应用
- 卸载后应用数据同步清除

**触发方式：** 管理平台下发 `uninstallApp` 指令，携带目标应用包名

### 4.5 恢复出厂设置（erase）

**指令效果：**
- 设备立即执行恢复出厂设置
- **所有数据将被清除，不可恢复**
- 设备重启后进入初始引导界面

**触发方式：** 管理平台下发 `erase` 指令

> ⚠️ 此操作需谨慎，设备数据将全部丢失。

### 4.6 WiFi控制（wifi）

| 指令参数 | 效果 |
|---------|------|
| `wifi` + params=1 | 关闭设备WiFi |
| `wifi` + params=0 | 开启设备WiFi |

### 4.7 蓝牙控制（bluetooth）

| 指令参数 | 效果 |
|---------|------|
| `bluetooth` + params=1 | 关闭蓝牙 |
| `bluetooth` + params=0 | 开启蓝牙 |

### 4.8 USB控制

| 指令类型 | 效果 |
|---------|------|
| `ManageUSB` / `openUsb` | 开启USB数据传输 |
| `closeUsb` | 关闭USB数据传输 |

> ⚠️ 关闭USB后，连接电脑将无法进行文件传输和调试。

### 4.9 OTG控制（ManageOTG）

| 指令参数 | 效果 |
|---------|------|
| params=1 | 禁用OTG外设连接 |
| params=0 | 启用OTG外设连接 |

### 4.10 NFC控制（ManageNFC）

| 指令参数 | 效果 |
|---------|------|
| params=1 | 禁用NFC功能 |
| params=0 | 启用NFC功能 |

### 4.11 截屏控制

| 指令类型 | 效果 |
|---------|------|
| `openScreenshot` | 允许截屏 |
| `closeScreenshot` / `DisableScreenCapture` | 禁止截屏 |

### 4.12 摄像头控制（setVideoDisabled）

| 指令参数 | 效果 |
|---------|------|
| params=1 | 禁用摄像头 |
| params=0 | 启用摄像头 |

### 4.13 麦克风控制（setMicrophoneDisabled）

| 指令参数 | 效果 |
|---------|------|
| params=1 | 禁用麦克风 |
| params=0 | 启用麦克风 |

### 4.14 移动数据控制

| 指令类型 | 效果 |
|---------|------|
| `turnOnMobiledata` | 强制开启移动数据 |
| `DisableInternet` + params=1 | 禁用数据网络连接 |

### 4.15 密码重置（newPassword）

**指令效果：**
- 设备解锁密码被强制修改为指定密码

**触发方式：** 管理平台下发 `newPassword` 指令，携带新密码参数

### 4.16 震动提醒

| 指令类型 | 效果 |
|---------|------|
| `startVibrationAlert` | 设备开始震动强提醒 |
| `stopVibrationAlert` | 停止震动提醒 |

### 4.17 机卡绑定（bindingSwitch）

| 指令参数 | 效果 |
|---------|------|
| params=1 | 开启机卡绑定校验（仅限绑定SIM卡使用） |
| params=0 | 关闭机卡绑定校验 |

## 5. 管理平台指令说明

### 5.1 MQTT指令格式

设备通过订阅MQTT主题接收管理平台指令。标准指令格式如下：

```json
{
  "topic": "operationName",
  "data": {
    "id": "command_001",
    "operationName": "lock",
    "params": "1"
  }
}
```

### 5.2 批量指令格式

多条指令可一次性下发，设备并行执行：

```json
{
  "topic": "operator",
  "data": [
    {"operationName": "lock", "params": "1"},
    {"operationName": "wifi", "params": "0"}
  ]
}
```

### 5.3 设备结果上报

设备执行完指令后，自动通过MQTT回传结果：

```json
{
  "deviceId": "设备序列号",
  "commandId": "command_001",
  "success": true,
  "message": "执行指令：设备锁屏"
}
```

### 5.4 策略下发

管理平台可下发设备策略（Policy），设备自动解析并应用：

| 策略类型 | 说明 |
|---------|------|
| `LimitPolicy` | 功能限制策略（截屏/多窗口/开发者模式等30余项） |
| `packagePolicy` | 应用管理策略（安装/运行/卸载黑白名单） |
| `WifiPolicy` | WiFi热点配置 |
| `bluetoothPolicy` | 蓝牙策略 |
| `ssidPolicy` | SSID白名单策略 |

## 6. 状态查看与监控

### 6.1 设备状态监控项

| 监控项 | 更新频率 | 说明 |
|--------|---------|------|
| 心跳 | 默认15分钟/次 | 设备在线状态 |
| SIM卡状态 | 实时 | ICCID、IMSI、卡槽状态 |
| 应用安装变化 | 实时 | 应用安装/卸载/更新事件 |
| 屏幕状态 | 实时 | 亮屏/熄屏切换 |
| 网络类型 | 按需 | WiFi/移动数据/无连接 |

### 6.2 心跳上报内容

心跳包包含以下设备信息：

```
- 设备序列号（Serial Number）
- IMEI
- IMSI
- ICCID
- 当前SIM卡数量
- 应用版本号
- 当前锁屏状态
- 机卡绑定状态
- 心跳时间戳
```

### 6.3 锁屏界面

设备被锁定后，显示FullScreenActivity全屏锁屏界面：

- 主文字（可自定义，默认"终端强制锁定，如需解锁请与管理员联系"）
- 副文字（可自定义）
- 设备信息（可选显示）

## 7. 常见问题与处理

### Q1：设备显示"服务未连接"

**原因：** MQTT服务未启动或连接中断

**处理方法：**
1. 检查网络是否正常
2. 重启应用或重启设备
3. 若持续无法连接，联系管理员检查服务器配置

---

### Q2：设备已锁定但无法解锁

**原因：** 解锁指令未送达或执行失败

**处理方法：**
1. 确保设备网络正常
2. 在管理平台确认解锁指令已发送
3. 检查设备日志获取详细错误信息

---

### Q3：应用无法安装

**原因：** 安装被黑名单拦截或权限不足

**处理方法：**
1. 检查该应用是否在安装黑名单中
2. 联系管理员调整应用策略
3. 确认设备存储空间充足

---

### Q4：设备管理员权限消失

**原因：** 设备被恢复出厂设置或被其他MDM应用覆盖

**处理方法：**
1. 重新激活设备管理员
2. 重新注册到管理平台
3. 若为设备被接管，需联系管理员确认归属

---

### Q5：机卡绑定校验失败

**原因：** 当前SIM卡ICCID与绑定记录不匹配

**处理方法：**
1. 确认设备使用的是正确的SIM卡
2. 联系管理员更新机卡绑定关系
3. 临时关闭机卡绑定功能需管理员下发 `bindingSwitch` + params=0

---

### Q6：熄屏后设备断开连接

**原因：** 系统省电策略或后台限制导致MQTT连接断开

**处理方法：**
1. 在设置中关闭省电模式
2. 将应用加入不受限制的后台应用白名单
3. 确保华为设备的"应用启动管理"中允许自启动和后台运行

---

### Q7：如何确认设备是否正常注册？

**确认方法：**
1. 通知栏显示"MQTT设备服务运行中"
2. 打开应用，主界面显示已连接状态
3. 在管理平台设备列表中查看设备状态为"在线"
