## 三、功能模块说明

### 3.1 通信模块（MqttService）

**文件路径**：`com.siyu.mdm.custom.device.service.MqttService`

**功能描述**：
- 作为Android前台Service运行，确保进程不被系统回收
- 通过MQTT协议与管理平台保持长连接
- 订阅设备私有命令主题，接收并执行下发指令
- 将命令执行结果通过MQTT异步回传至管理平台
- 支持设备配置更新（MQTT服务器地址动态变更）
- Android 13+ 适配通知权限，Android 14+ 适配前台服务类型

**核心订阅主题**：
| 主题 | 用途 |
|------|------|
| `device/{deviceId}/command` | 接收设备私有控制指令 |
| `device/{deviceId}/config` | 接收设备配置更新 |
| `device/will` | 设备状态上报与命令结果回传 |

---

### 3.2 命令调度模块（MdmCommandManager）

**文件路径**：`com.siyu.mdm.custom.device.util.MdmCommandManager`

**功能描述**：
- 统一命令执行入口，支持设备控制指令和业务指令两类
- 设备控制指令通过`operationName` + `params`参数驱动
- 支持批量命令并行执行与结果汇总
- 命令执行超时机制（默认30秒）
- 命令执行结果关联命令ID，便于管理平台追踪

**支持的控制指令列表**：

| 指令类型 | operationName | 功能说明 |
|----------|--------------|---------|
| 设备锁定 | `lock` / `unlock` | 设备屏幕锁定与解锁 |
| 设备擦除 | `erase` | 恢复出厂设置 |
| 应用安装 | `installUpdateApp` | 远程安装或更新APK应用 |
| 应用卸载 | `uninstallApp` | 远程卸载指定应用 |
| WiFi控制 | `wifi` | 开启或关闭WiFi |
| WiFi热点 | `openWifiAp` / `closeWifiAp` | 开启或关闭WiFi热点 |
| 蓝牙控制 | `bluetooth` | 开启或关闭蓝牙 |
| USB控制 | `ManageUSB` / `openUsb` / `closeUsb` | USB数据连接管理 |
| OTG控制 | `ManageOTG` | OTG外设连接管理 |
| NFC控制 | `ManageNFC` | NFC功能管理 |
| 截屏控制 | `openScreenshot` / `closeScreenshot` | 系统截屏功能开关 |
| 麦克风禁用 | `setMicrophoneDisabled` | 禁用或恢复麦克风 |
| 摄像头禁用 | `setVideoDisabled` | 禁用或恢复摄像头 |
| 移动数据 | `turnOnMobiledata` | 强制开启或关闭移动数据 |
| 数据连接 | `DisableInternet` | 禁用或启用数据网络 |
| 密码重置 | `newPassword` | 重置设备解锁密码 |
| APN配置 | `SetApnOrder` | 配置APN接入点 |
| 机卡绑定 | `bindingSwitch` | 开启/关闭机卡绑定校验 |
| 震动提醒 | `startVibrationAlert` / `stopVibrationAlert` | 远程触发或停止设备震动提醒 |

### 3.3 华为MDM接口封装（HuaweiMDMManager）

**文件路径**：`com.siyu.mdm.custom.device.util.HuaweiMDMManager`

**功能描述**：
- 实现了`HuaweiMDM`复合接口，封装华为MDM SDK全部设备管理能力
- 通过`DevicePackageManager`、`DeviceCameraManager`、`DeviceBluetoothManager`等各类管理器操作设备
- 提供管理员权限检查、应用黑白名单管理、硬件功能限制等核心能力
- 引入反射机制安全调用`DeviceRestrictionManager`，保证设备兼容性
- 支持水印服务（无障碍服务 Overlay）启动与管理

**接口结构**：

```
HuaweiMDM（复合接口）
├── MDMBase            — 管理员组件与权限校验
├── AppManagement      — 应用安装/卸载白黑名单管理
├── HardwareRestriction — 摄像头/麦克风/WiFi/蓝牙/USB/NFC等硬件控制
├── NetworkManagement  — 移动数据/APN/连接始终在线管理
├── SecurityManagement — 密码重置/设备锁屏/恢复出厂设置
└── PrivacyProtection  — 隐私保护（预留）
```

### 3.4 设备策略模块（Policy）

**文件路径**：`com.siyu.mdm.custom.device.json.Policy`

**功能描述**：
- 解析服务器下发的多类型策略JSON

- `PackagePolicy`控制应用安装/运行/卸载策略
- `WifiPolicy`存储WiFi热点配置列表

## 五、MQTT通信协议

### 5.1 消息格式

所有MQTT消息采用JSON格式，通用结构如下：

**下发指令格式**：
```json
{
  "topic": "operationName",
  "data": {
    "id": "command_001",
    "operationName": "lock",
    "params": "1"
  }
}
```

**批量指令格式**：
```json
{
  "topic": "operator",
  "data": [
    {"operationName": "lock", "params": "1"},
    {"operationName": "wifi", "params": "0"}
  ]
}
```

**命令结果上报**：
```json
{
  "deviceId": "xxx",
  "commandId": "command_001",
  "success": true,
  "message": "执行指令：设备锁屏"
}
```

### 5.2 心跳机制

- 设备定期向服务器发送心跳包（`heartbeat/request`）
- 心跳内容包含设备基本信息（IMEI、IMSI、ICCID、SIM数量、版本号等）
- 服务器响应后更新设备在线状态

## 七、关键业务流程

### 7.1 设备注册与激活流程

```
1. 设备开机 → BootReceiver接收BOOT_COMPLETED广播
2. 启动MqttService前台服务
3. 连接MQTT服务器
4. 订阅设备私有命令主题
5. 发布设备注册消息（包含设备基本信息）
6. 管理平台记录设备在线状态
```

### 7.2 远程控制命令执行流程

```
1. 管理平台通过MQTT下发控制指令
2. MqttService接收并解析JSON消息
3. MdmCommandManager根据operationName分发至对应处理方法
4. HuaweiMDMManager调用华为MDM SDK执行实际操作
5. 执行结果封装为CommandResult
6. 通过MQTT将结果上报至管理平台
```

### 7.3 策略更新流程

```
1. 管理平台下发策略数据（Policy JSON）
2. 客户端解析policyType确定策略类型
3. 根据类型调用HuaweiMDMManager相应接口
4. 策略内容应用到设备系统
5. 变更结果通过心跳或即时消息反馈
```

## 九、兼容性说明

- **最低Android版本**：7.0（Nougat，API 24）
- **推荐Android版本**：10.0 ～ 14.0
- **华为设备适配**：支持华为EMUI 8.0及以上设备
- **MDM能力**：需要设备支持华为Device Admin企业管理接口
- **Android 13+**：需要运行时申请`POST_NOTIFICATIONS`通知权限
- **Android 14+**：前台服务必须指定`foregroundServiceType`为`connectedDevice`

## 十、技术特点总结

1. **模块化设计**：通信层、命令层、接口层、执行层分离，职责清晰
2. **华为MDM深度整合**：充分利用华为设备管理企业API，支持30余项设备控制能力
3. **MQTT轻量通信**：低开销、长连接，适合设备管理场景
4. **前台服务保活**：确保设备管理服务不被系统Kill
5. **命令可追溯**：每条命令携带唯一ID，支持批量命令执行与结果汇总
6. **机卡绑定安全**：支持SIM卡与设备绑定校验，防止非法设备接入
7. **远程震动提醒**：熄屏状态下仍可触发设备提醒
8. **代码规模适中**：约5,100行核心代码，覆盖设备管理完整生命周期
