## 1. 软件概述

MDM企业设备管理系统（设备端）是部署在Android设备上的客户端程序，配合MDM管理平台实现对企业移动设备的**远程集中管控**。

**主要功能：**

- 接收并执行管理平台下发的远程控制指令
- 设备状态监控与心跳保活
- 应用安装、卸载、黑白名单管控
- 硬件功能（WiFi/蓝牙/USB/摄像头/麦克风等）远程控制
- 设备锁定、解锁、恢复出厂设置
- 设备重启、密码重置
- 多厂商自动适配（华为/小米/vivo）

## 2. 安装与激活

### 2.1 安装前准备

- 设备需为Android 7.0（API 24）及以上版本
- 设备需支持MDM能力（推荐使用华为、小米、vivo主流设备）
- 获得管理平台分配的设备注册信息
- APK安装包（由管理员提供）

### 2.2 安装步骤

**Step 1：获取安装包**

从管理员处获取APK安装包，拷贝至设备存储。

**Step 2：允许安装未知来源应用**

> 设置 → 安全与隐私 → 更多设置 → 外部来源应用下载 → 允许

> ⚠️ 若设备已受MDM管控，此步骤可能由管理员预置，无需手动操作。

**Step 3：安装APK**

找到APK文件，点击安装，按提示完成安装流程。

**Step 4：激活设备管理员**

首次打开应用时，系统会提示"激活设备管理员"，选择**允许**。

> ⚠️ 设备管理员权限是MDM核心能力的前提，不激活则无法使用远程管控功能。

**Step 5：等待注册成功**

应用激活后自动连接MQTT服务器并完成设备注册。注册成功后通知栏显示"MQTT设备服务运行中"。

### 2.3 激活失败处理

| 现象 | 原因 | 处理方法 |
|------|------|---------|
| 无法激活设备管理员 | 设备已有其他MDM应用 | 移除旧管理应用或恢复出厂设置 |
| 注册连接失败 | 网络不通 | 检查WiFi或移动数据是否正常 |
| 显示"连接超时" | MQTT服务器地址错误 | 联系管理员确认服务器配置 |
| 应用频繁退出 | 权限不足 | 检查通知权限、前台服务权限是否授予 |

## 3. 主界面介绍

### 3.1 主界面（MainActivity）

主界面用于展示当前设备基本信息与连接状态：

| 显示区域 | 内容 |
|---------|------|
| 设备信息 | 设备型号、厂商、操作系统版本 |
| 连接状态 | MQTT连接状态（已连接/重连中/未连接） |
| 厂商标识 | 当前设备厂商（华为/小米/vivo） |
| 版本信息 | 应用版本号 |

### 3.2 设备管理界面（MdmManagementActivity）

提供MDM各项功能的快捷操作入口，包括设备锁定、应用管理、网络控制等常用功能的状态展示。

### 3.3 全屏锁屏界面（FullScreenActivity）

设备被管理员锁定后自动弹出，显示管理员自定义提示信息：

- 主文字（可自定义，默认："设备已锁定"）
- 副文字（可自定义，默认："请联系管理员"）
- 无法退出，需管理员下发解锁指令

### 3.4 USB绑定界面（USBBindActivity）

管理USB设备绑定关系，显示当前已绑定的USB设备列表及绑定状态。

## 4. 设备控制操作

以下操作通过管理平台远程下发指令执行，设备端无需人工干预。

### 4.1 设备锁定（lock）

**效果：** 设备立即进入全屏锁屏状态，显示管理员预设文字，无法正常使用。

**恢复方式：** 管理平台下发 `unlock` 解锁指令。

### 4.2 设备解锁（unlock）

**效果：** 退出锁屏模式，恢复正常设备使用。

### 4.3 恢复出厂设置（wipe）

**效果：** 设备立即执行恢复出厂设置，**所有数据将被清除**。

> ⚠️ 此操作不可逆，设备数据将全部丢失，请管理员谨慎使用。

### 4.4 设备重启（reboot）

**效果：** 设备立即重启。

### 4.5 密码重置（reset_password）

**效果：** 将设备解锁密码强制修改为指定密码。

**参数：** 携带新密码参数

### 4.6 应用安装（install_app）

**效果：** 设备自动下载指定APK并静默安装。

**参数：** APK下载URL地址（`apkPath`）

### 4.7 应用卸载（uninstall_app）

**效果：** 远程卸载指定应用及其数据。

**参数：** 目标应用包名（`packageName`）

### 4.8 WiFi控制

| 指令 | 效果 |
|------|------|
| `disable_wifi` | 关闭设备WiFi |
| `enable_wifi` | 开启设备WiFi |

### 4.9 移动数据控制

| 指令 | 效果 |
|------|------|
| `disable_mobile_data` | 禁用移动数据连接 |
| `enable_mobile_data` | 启用移动数据连接 |

### 4.10 蓝牙控制

| 指令 | 效果 |
|------|------|
| `set_bluetooth` + enabled=false | 关闭蓝牙 |
| `set_bluetooth` + enabled=true | 开启蓝牙 |

### 4.11 GPS控制

| 指令 | 效果 |
|------|------|
| `set_gps` + enabled=false | 关闭GPS定位 |
| `set_gps` + enabled=true | 开启GPS定位 |

### 4.12 截屏控制

| 指令 | 效果 |
|------|------|
| `disable_screenshot` | 禁止系统截屏 |
| `enable_screenshot` | 允许系统截屏 |

### 4.13 摄像头控制

| 指令 | 效果 |
|------|------|
| `disable_camera` | 禁用摄像头 |
| `enable_camera` | 启用摄像头 |

### 4.14 麦克风控制

| 指令 | 效果 |
|------|------|
| `disable_microphone` | 禁用麦克风 |
| `enable_microphone` | 启用麦克风 |

## 5. 管理平台指令说明

### 5.1 MQTT命令格式

设备通过订阅MQTT主题接收管理平台指令：

**标准命令格式**：
```json
{
  "commandId": "uuid-123456",
  "action": "lock",
  "params": {
    "mainText": "设备已锁定",
    "subText": "请联系管理员"
  },
  "timestamp": 1704067200000
}
```

**查询类命令**：
```json
{
  "commandId": "uuid-789",
  "action": "get_device_info"
}
```

### 5.2 支持的命令列表

| action | 类型 | 说明 | 参数 |
|--------|------|------|------|
| `lock` | 控制 | 锁定设备 | mainText, subText |
| `unlock` | 控制 | 解锁设备 | - |
| `wipe` | 控制 | 恢复出厂设置 | - |
| `reboot` | 控制 | 重启设备 | - |
| `reset_password` | 控制 | 重置密码 | password |
| `install_app` | 控制 | 安装应用 | apkPath |
| `uninstall_app` | 控制 | 卸载应用 | packageName |
| `disable_wifi` | 控制 | 关闭WiFi | - |
| `enable_wifi` | 控制 | 开启WiFi | - |
| `disable_mobile_data` | 控制 | 关闭移动数据 | - |
| `enable_mobile_data` | 控制 | 开启移动数据 | - |
| `set_gps` | 控制 | 设置GPS | enabled |
| `set_bluetooth` | 控制 | 设置蓝牙 | enabled |
| `disable_screenshot` | 控制 | 禁止截屏 | - |
| `enable_screenshot` | 控制 | 允许截屏 | - |
| `disable_camera` | 控制 | 禁用摄像头 | - |
| `enable_camera` | 控制 | 启用摄像头 | - |
| `disable_microphone` | 控制 | 禁用麦克风 | - |
| `enable_microphone` | 控制 | 启用麦克风 | - |
| `get_device_info` | 查询 | 获取设备信息 | - |
| `get_app_list` | 查询 | 获取已安装应用 | - |

### 5.3 命令结果上报

设备执行完指令后自动通过MQTT回传结果：

```json
{
  "commandId": "uuid-123456",
  "action": "lock",
  "success": true,
  "message": "设备已锁定",
  "timestamp": 1704067201000,
  "data": {}
}
```

### 5.4 遗嘱消息（设备异常离线）

设备异常断开连接时，MQTT自动发布遗嘱消息：

```json
{
  "action": "offline",
  "code": "设备序列号"
}
```

## 6. 状态监控与心跳

### 6.1 心跳机制

设备按配置周期（默认60秒）向服务器发送心跳包，维持设备注册状态。

**心跳间隔策略**：
| 场景 | 间隔 | 说明 |
|------|------|------|
| 正常模式 | 60秒 | 常规心跳 |
| 熄屏模式 | 适当延长 | 省电策略 |
| 低电量（<20%） | 延长至5分钟 | 极端省电 |
| 充电中 + 电量>50% | 缩短至1分钟 | 高性能模式 |

**心跳上报内容**：
- 设备ID（序列号）
- 电池电量与充电状态
- 屏幕状态（亮屏/熄屏）
- 网络类型（WiFi/移动数据）
- 时间戳

### 6.2 应用变更监听

设备自动监听应用安装/卸载/更新事件并即时上报管理平台，便于管理员掌握设备软件环境变化。

### 6.3 屏幕状态监听

设备屏幕亮灭变化时触发事件，用于协调MQTT连接策略和心跳频率调整。

## 8. 常见问题与处理

### Q1：设备显示"服务未连接"

**原因：** MQTT服务未启动或连接中断

**处理方法：**
1. 检查网络是否正常（WiFi或移动数据）
2. 重启应用或重启设备
3. 若持续无法连接，联系管理员检查MQTT服务器配置

---

### Q2：设备已锁定但无法解锁

**原因：** 解锁指令未送达或执行失败

**处理方法：**
1. 确保设备网络正常（锁屏状态下仍可接收MQTT消息）
2. 在管理平台确认解锁指令已发送
3. 检查设备日志获取详细错误信息

---

### Q3：应用安装失败

**原因：** 安装被策略拦截、APK地址无效或存储空间不足

**处理方法：**
1. 检查该应用是否在安装黑名单中
2. 确认APK下载地址有效且设备可访问
3. 确认设备存储空间充足
4. 联系管理员调整应用管控策略

---

### Q4：设备管理员权限消失

**原因：** 设备被恢复出厂设置或被其他MDM应用覆盖

**处理方法：**
1. 重新激活设备管理员
2. 重新注册到管理平台
3. 若设备被其他MDM应用接管，需联系原管理员确认归属

---

### Q5：部分指令在设备上不生效

**原因：** 该设备厂商可能不支持特定API

**处理方法：**
1. 检查设备厂商是否在支持列表中（华为/小米/vivo）
2. 使用诊断工具获取详细错误报告
3. 联系开发人员排查设备兼容性问题

---

### Q6：熄屏后设备断开连接

**原因：** 系统省电策略或后台限制导致MQTT连接被强制断开

**处理方法：**
1. 在设置中关闭省电模式
2. 将应用加入不受限制的后台应用白名单
3. 在华为/小米/vivo设备的"应用启动管理"中允许自启动和后台运行

---

### Q7：如何确认设备是否正常注册？

**确认方法：**
1. 通知栏显示"MQTT设备服务运行中"
2. 打开应用，主界面显示"已连接"状态
3. 在管理平台设备列表中查看设备状态为"在线"

---

### Q8：恢复出厂设置后设备还能被管理吗？

**说明：** 恢复出厂设置会清除设备上所有数据包括MDM应用和注册信息。设备将恢复为全新状态，需要重新安装应用并完成注册才能继续被管理。
