## 一、产品概述

MDM 企业移动管理系统（鸿蒙 NEXT 6.0+）是上海思御信息科技有限公司自主研发的企业级移动设备管控应用，运行于华为鸿蒙 NEXT 6.0 及以上操作系统设备上，通过 MQTT 协议与服务器建立长连接，实现企业移动设备的远程集中管控。

系统提供设备远程锁定、应用管理、网络与外设管控、设备信息查询等功能，适用于政企单位对移动设备的合规安全管理需求。

**当前运行状态**：Mock 模式（模拟执行，非真实MDM能力）

## 二、安装与初始化

### 2.1 编译构建

1. 使用 DevEco Studio 打开项目根目录 `mdm-enterprise-hos`
2. 选择 HarmonyOS 设备或模拟器
3. 点击运行（Run）按钮，DevEco Studio 自动编译并安装到设备

### 2.2 首次启动

首次打开应用时，系统自动完成以下初始化操作：

1. **ConfigStore 初始化**：在 Preferences 中创建 `mdm_config` 存储实例
2. **设备ID生成**：若为首次启动，系统自动生成设备ID（格式：`mdm_{时间戳36}_{随机字符}`）并持久化存储
3. **MQTT后台服务启动**：系统自动启动后台长连接服务，尝试连接 MQTT Broker `tcp://192.168.1.199:1883`
4. **主页加载**：加载设备状态展示页面

## 三、主页面功能说明

主页（Index.ets）展示设备实时状态，并提供命令测试功能。

### 3.1 设备状态卡片

页面顶部卡片实时显示以下信息：

| 显示项 | 说明 |
|--------|------|
| 设备ID | 设备的唯一标识符，用于MQTT通信 |
| 连接状态 | MQTT连接状态：已连接/连接中/重连中/未连接/错误 |
| 电池 | 电量百分比及充电状态（⚡表示正在充电） |
| 网络 | 当前网络类型：WiFi / 移动数据 / 无网络 |
| 心跳模式 | 当前心跳间隔策略：高性能/正常/省电/极致省电 |

设备状态每2秒自动刷新一次。

### 3.2 MQTT命令测试区域

在 Mock 模式下，可点击页面中的按钮发送测试命令，验证 MQTT 通信链路和命令处理流程。

**可用测试命令**：

| 按钮 | 发送的action | 说明 |
|------|------------|------|
| 锁定设备 | lock | 锁定设备屏幕 |
| 解锁设备 | unlock | 解锁屏幕 |
| 重启设备 | reboot | 重启设备 |
| 获取设备信息 | get_device_info | 查询设备基本信息 |
| 禁用WiFi | disable_wifi | 禁用WiFi网络 |
| 启用WiFi | enable_wifi | 启用WiFi网络 |
| 禁用相机 | disable_camera | 禁用相机功能 |
| 启用相机 | enable_camera | 启用相机功能 |

点击按钮后，命令日志区域显示发送时间和执行结果。日志格式为 `[时间戳] 发送命令: xxx` 和 `[时间戳] 执行结果: xxx`。

### 3.3 MDM功能说明

页面底部显示当前 MDM 运行模式说明。系统当前运行在 Mock 模式下，所有命令均为模拟执行，不执行真实的设备管控操作。

## 五、远程管控命令说明

以下命令由服务器通过 MQTT 下发到设备，设备自动接收并执行（Mock模式下模拟执行）。

### 5.1 设备控制命令

| action | 功能 | 参数 |
|--------|------|------|
| lock | 锁定设备屏幕，显示提示文字 | `mainText`（主提示）、`subText`（副提示） |
| unlock | 解锁屏幕 | 无 |
| wipe | 恢复出厂设置（清除所有数据） | 无 |
| reboot | 重启设备 | 无 |
| reset_password | 重置设备解锁密码 | `password`（新密码） |

**锁定命令示例**：
```json
{
  "commandId": "cmd-lock-001",
  "action": "lock",
  "params": {
    "mainText": "设备已锁定",
    "subText": "请联系管理员：400-xxx-xxxx"
  },
  "timestamp": 1704067200000
}
```

**重置密码命令示例**：
```json
{
  "commandId": "cmd-pwd-001",
  "action": "reset_password",
  "params": {
    "password": "Admin@1234"
  },
  "timestamp": 1704067200000
}
```

### 5.2 应用管理命令

| action | 功能 | 参数 |
|--------|------|------|
| install_app | 安装应用 | `apkPath`（APK文件路径） |
| uninstall_app | 卸载应用 | `bundleName`（应用包名） |
| get_app_list | 获取已安装应用列表 | 无 |

### 5.3 网络管控命令

| action | 功能 |
|--------|------|
| disable_wifi | 禁用WiFi |
| enable_wifi | 启用WiFi |
| disable_mobile_data | 禁用移动数据 |
| enable_mobile_data | 启用移动数据 |
| set_gps | GPS开关（参数：`enabled: true/false`） |
| set_bluetooth | 蓝牙开关（参数：`enabled: true/false`） |

### 5.4 外设管控命令

| action | 功能 |
|--------|------|
| disable_camera | 禁用相机 |
| enable_camera | 启用相机 |
| disable_microphone | 禁用麦克风 |
| enable_microphone | 启用麦克风 |
| disable_usb_debug | 禁用USB调试 |
| enable_usb_debug | 启用USB调试 |
| disable_screenshot | 禁用截屏 |
| enable_screenshot | 启用截屏 |

### 5.5 信息查询命令

## 六、心跳与在线机制

### 6.1 心跳上报

设备按动态调整的间隔向服务器发送心跳消息，包含以下数据：

```json
{
  "deviceId": "mdm_xxxxxx_xxxxxx",
  "batteryLevel": 85,
  "isCharging": false,
  "isScreenOn": true,
  "networkType": "WiFi",
  "heartbeatMode": "normal",
  "timestamp": 1704067200000
}
```

### 6.2 智能心跳间隔

| 当前状态 | 心跳间隔 | 模式说明 |
|---------|---------|---------|
| 充电中且电量>50% | 60秒 | 高性能模式 |
| 正常使用 | 120秒 | 正常模式 |
| 电量<20%且未充电 | 300秒 | 省电模式 |
| 息屏且未充电 | 600秒 | 极致省电模式 |

### 6.3 离线通知

当设备与 MQTT Broker 的连接断开时（无论主动还是异常断开），Broker 自动向 `mdm/report` 主题发布设备的离线通知（Will Message），服务器据此将设备状态标记为离线。

### 6.4 自动重连

断线后设备自动尝试重连，重连间隔根据断开时长智能调整：5秒 → 30秒 → 60秒 → 180秒 → 300秒（渐进延长）。

## 七、Mock 模式说明

### 7.1 什么是 Mock 模式

Mock 模式是一种开发阶段的替代实现模式。在获取华为企业 MDM 证书之前，系统使用 Mock 实现（模拟对象）替代真实的 MDM Kit 调用，所有命令执行结果为预设的模拟响应，不执行真实的设备管控操作。

### 7.2 如何判断当前模式

主页 MDM 功能说明区域会显示"当前运行在 Mock 模式下，所有MDM命令均为模拟执行"的橙色提示。

### 7.3 切换到真实 MDM 能力

1. 向华为申请企业 MDM 证书并获得签名证书（.p7b）
2. 在 DevEco Studio 中配置签名证书
3. 打开 `entry/src/main/ets/mdm/MdmApiFactory.ets` 文件
4. 将 `USE_MOCK = true` 改为 `USE_MOCK = false`
5. 取消注释真实实现的实例化代码
6. 重新编译运行
